# DEV_TEST_PLAN.md — Scratch Editor MVP 수동 회귀 테스트

## 1) 목적 / 범위

이 문서는 Scratch Editor MVP(M0~M6)의 **수동 회귀 테스트 게이트**다.  
회귀 체크리스트(§13) 10개 항목 + 스트레스 테스트(§12)를 모두 PASS해야 “릴리즈 가능”으로 간주한다.

전제/제약:
- MVP의 커서/선택 인덱스는 **바이트 인덱스(ASCII 기준)** 을 전제로 한다.
- IME(한글 조합 등)는 MVP 범위 밖이다.
- `USE_SCRATCH_EDITOR` 기본값은 **Off**(기존 QuickNotes 보호)다.

## 2) 테스트 준비

### 환경
- Don’t Starve Together 실행
- QuickNotes 모드 활성화

### Mod Config (테스트 중 자주 바꾸는 옵션)
- `Use Scratch Editor (Experimental)` (`USE_SCRATCH_EDITOR`)
  - Off: 기존 QuickNotes(TextEdit 기반) 경로
  - On: Scratch Editor(WIP) 경로
- `Debug Editor HUD` (`DEBUG_EDITOR`)
  - Off: 디버그 HUD 숨김
  - On: caret/selection/viewport/hit test HUD 표시 (Scratch Editor 전용)

### 권장 테스트 데이터
- **200~500줄** 텍스트: 짧은 줄/긴 줄 혼합(스크롤/선택/성능)
- **긴 한 줄(1000자)**: 클릭 커서 이동/폭 측정 성능 관찰

## 3) 회귀 테스트 체크리스트 (§13 — 10개 항목)

> 각 항목은 **절차 / 기대 결과 / 실패 시 메모**를 반드시 기록한다.

### 1. **N으로 열기**
- [ ] 절차: 다양한 상황(인게임 HUD 열림/닫힘 등)에서 `N`으로 노트 열기 시도
- [ ] 기대 결과: 규칙대로 열리며, 입력 포커스 충돌 시에는 열리지 않는다(기존 정책 유지)
- [ ] 실패 시 메모:

### 2. **입력**
- [ ] 절차: 일반 문자 입력, Enter, Backspace/Delete
- [ ] 기대 결과: 입력이 누락/중복 없이 반영된다
- [ ] 실패 시 메모:

### 3. **이동**
- [ ] 절차: ←→↑↓, Home/End, PageUp/PageDown
- [ ] 기대 결과: 커서 이동이 논리적으로 일관되고, 문서 경계에서 안전하게 클램프된다
- [ ] 실패 시 메모:

### 4. **선택**
- [ ] 절차: Shift 선택(키보드), 마우스 드래그 선택 + 화면 밖 드래그 자동 스크롤
- [ ] 기대 결과: 선택 범위가 끊기지 않고, 드래그 중 포커스가 튀지 않는다
- [ ] 실패 시 메모:

### 5. **클릭 커서 이동**
- [ ] 절차: 스크롤된 상태에서도 여러 위치 클릭으로 커서 이동
- [ ] 기대 결과: 클릭한 위치(행/열)에 커서가 합리적으로 배치된다
- [ ] 실패 시 메모:

### 6. **스크롤**
- [ ] 절차: 휠, 스크롤바 thumb, 트랙 클릭(점프)
- [ ] 기대 결과: 모든 스크롤 소스가 동일한 viewport를 갱신하며 동기화가 깨지지 않는다
- [ ] 실패 시 메모:

### 7. **자동 스크롤**
- [ ] 절차: 타이핑/이동으로 커서가 화면 밖으로 나가도록 만든 뒤 입력 지속
- [ ] 기대 결과: 커서가 화면 밖으로 나가지 않도록 자동 스크롤이 동작한다
- [ ] 실패 시 메모:

### 8. **창 UX**
- [ ] 절차: 타이틀 드래그, 닫기 버튼, 바깥 클릭 닫기 — 에디터 클릭/드래그를 방해하지 않는지 확인
- [ ] 기대 결과: 창 UX가 편집 UX와 충돌하지 않는다(드래그/클릭/닫기 동작이 예측 가능)
- [ ] 실패 시 메모:

### 9. **저장**
- [ ] 절차: 닫았다 열기, 월드 이동/재접속 등 후 내용 유지 확인
- [ ] 기대 결과: 내용이 유지되며, 중복 저장/중복 백업이 과도하게 발생하지 않는다
- [ ] 실패 시 메모:

### 10. **리소스/누수**
- [ ] 절차: 여러 번 열고 닫기(최소 10회), 드래그/선택/스크롤을 섞어서 반복
- [ ] 기대 결과: blink task/drag lock 등 리소스가 닫을 때 정리되고, 성능이 누적 악화되지 않는다
- [ ] 실패 시 메모:

## 4) 스트레스 테스트 (§12)

### A) 200~500줄 문서 스트레스
- [ ] PageDown 연타(20회 이상)
- [ ] 스크롤바 드래그 왕복(상단↔하단 5회)
- [ ] 드래그 선택 + 자동 스크롤(화면 밖까지 끌고 가기)
- [ ] 선택 후 Delete/Backspace

관찰 포인트:
- [ ] 프레임 흔들림/입력 지연
- [ ] 커서/선택 오작동
- [ ] 스크롤 동기화 깨짐(thumb/viewport 불일치)

### B) 긴 한 줄(1000자) 클릭 커서 이동
- [ ] 한 줄에 1000자 입력(또는 붙여넣기)
- [ ] 서로 다른 x 좌표로 클릭을 반복(20회 이상)

관찰 포인트:
- [ ] 클릭 반응 지연(눈에 띄는 프레임 드랍)
- [ ] col 계산이 비합리적으로 튀는지(히트테스트 정확도)
- [ ] 필요 시: X→col 최적화(측정 위젯 재사용/이분탐색) 개선이 필요한지
